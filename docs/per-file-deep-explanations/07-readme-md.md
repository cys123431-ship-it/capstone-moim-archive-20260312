# README.md 깊은 해설서

## 1. 파일 개요
- 파일명: `README.md`
- 역할: 저장소 첫 화면에서 프로젝트를 소개하는 문서다. 기능, 파일 구성, 실행 방법, 배포 주소를 짧게 요약한다.
- 사용 시점: GitHub 저장소를 열었을 때 가장 먼저 읽히는 안내 문서다.
- 이 파일에서 들여쓰기 읽는 법: Markdown은 제목은 바깥에 두고, 목록이나 코드 블록처럼 종속된 내용은 필요할 때만 안쪽으로 들여쓴다.
- 기본 연결 대상:
- 프로젝트의 핵심 파일들을 한 줄씩 소개한다.
- GitHub Pages 배포 주소를 적어 결과물 접근 경로를 알려 준다.
- 소스 코드를 직접 읽기 전에 전체 목적을 빠르게 파악하도록 돕는다.
