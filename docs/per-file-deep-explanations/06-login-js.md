# login.js 깊은 해설서

## 1. 파일 개요
- 파일명: `login.js`
- 역할: 로그인 데모 페이지의 간단한 JavaScript 파일이다. 실제 인증 대신 안내 메시지를 보여 주고, 비밀번호 보기/숨기기만 처리한다.
- 사용 시점: `login.html`이 열린 뒤 `defer`로 실행된다.
- 이 파일에서 들여쓰기 읽는 법: JavaScript는 함수, 조건문, 반복문, 객체, 배열 안으로 들어갈수록 들여쓰기가 깊어진다. 즉 들여쓰기는 '지금 이 줄이 어느 블록 안에 속하는가'를 보여 주는 표시다.
- 기본 연결 대상:
- `login.html`의 `#loginForm`, `#loginMessage`, `#passwordInput`, `#togglePasswordButton`, `#googleButton`을 선택한다.
- 화면 스타일은 `login.css`가 맡고, 이 파일은 클릭과 제출 반응만 맡는다.
- 실제 API는 아직 없으므로 서버 통신 대신 사용자 안내 문구만 갱신한다.

~~~~js
   2 | const loginMessage = document.querySelector("#loginMessage");
   3 | const passwordInput = document.querySelector("#passwordInput");
   4 | const togglePasswordButton = document.querySelector("#togglePasswordButton");
   5 | const googleButton = document.querySelector("#googleButton");
   6 | 
   7 | // 실제 인증 대신, 현재 단계가 UI 데모라는 점을 같은 위치에 안내한다.
   8 | function showLoginMessage(message) {
   9 |   loginMessage.textContent = message;
  10 | }
  11 | 
  12 | // 비밀번호 칸의 type만 바꿔 가장 기본적인 보기/숨기기 흐름을 만든다.
  13 | function togglePasswordVisibility() {
  14 |   const isPassword = passwordInput.type === "password";
  15 |   passwordInput.type = isPassword ? "text" : "password";
  16 |   togglePasswordButton.setAttribute(
  17 |     "aria-label",
  18 |     isPassword ? "비밀번호 숨기기" : "비밀번호 보기"
  19 |   );
  20 | }
  21 | 
  22 | // 지금 단계는 API 제외 범위라서 제출 시 실제 로그인 대신 안내 문구만 보여 준다.
  23 | loginForm.addEventListener("submit", (event) => {
  24 |   event.preventDefault();
  25 | 
  26 |   if (!loginForm.reportValidity()) {
  27 |     return;
  28 |   }
  29 | 
  30 |   showLoginMessage(
  31 |     "지금은 로그인 UI만 구현된 상태입니다. 실제 인증 기능은 아직 연결하지 않았습니다."
  32 |   );
  33 | });
  34 | 
  35 | togglePasswordButton.addEventListener("click", togglePasswordVisibility);
  36 | 
  37 | googleButton.addEventListener("click", () => {
  38 |   showLoginMessage(
  39 |     "구글 로그인 버튼도 현재는 디자인 데모 단계입니다. 실제 OAuth 연결은 나중에 붙이면 됩니다."
  40 |   );
  41 | });
~~~~

- 블록 원문:
~~~~js
   8 | function showLoginMessage(message) {
   9 |   loginMessage.textContent = message;
  10 | }
  11 | 
  12 | // 비밀번호 칸의 type만 바꿔 가장 기본적인 보기/숨기기 흐름을 만든다.
~~~~

## 6. 이 파일은 어떤 것과 연결되는가
- JS가 선택하는 HTML 요소: `#loginForm`, `#loginMessage`, `#passwordInput`, `#togglePasswordButton`, `#googleButton`.
- HTML ↔ CSS 연결 보조: 이 파일은 시각 스타일을 바꾸지 않고, `login.css`로 꾸며진 요소에 메시지와 클릭 반응만 준다.
- asset 참조 여부: 이미지 자산을 사용하지 않는다.
- 상대경로 사용: 페이지 이동을 직접 처리하지는 않지만 `login.html` 안에서 상대경로로 연결되어 실행된다.
