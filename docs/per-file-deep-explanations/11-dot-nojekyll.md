# .nojekyll 깊은 해설서

## 1. 파일 개요
- 파일명: `.nojekyll`
- 역할: GitHub Pages가 저장소 파일을 Jekyll 처리 없이 그대로 배포하도록 알려 주는 빈 파일이다.
- 사용 시점: GitHub Pages 배포 시 정적 파일을 그대로 서빙하고 싶을 때 의미가 생긴다.
- 이 파일에서 들여쓰기 읽는 법: 빈 파일이라 들여쓰기 구조는 없다.
- 기본 연결 대상:
- `README.md`와 `docs/web-implementation-guide.md`에서 배포 맥락과 함께 언급된다.
- 소스 코드 실행 로직에는 직접 관여하지 않지만 배포 환경에는 영향을 준다.
- 빈 파일이지만 배포 설정 의도를 드러내는 메타 파일이다.

## 6. 이 파일은 어떤 것과 연결되는가
- HTML/CSS/JS 연결: 직접 연결은 없다.
- 배포 연결: GitHub Pages가 Jekyll을 거치지 않고 정적 파일을 그대로 배포하게 만드는 메타 파일이다.
- 상대경로 사용: 경로가 아니라 파일 존재 자체가 의미를 가진다.
