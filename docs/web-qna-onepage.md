# 웹 설명 암기용 한 장

## 1. 제일 먼저 말할 한 줄

`저는 원래 웹이 처음이라 HTML도 거의 모르는 상태였고, 그래서 AI를 튜터처럼 활용해서 구조를 배우면서 웹 파트를 정리했습니다.`

---

## 2. 30초 답변

`저는 원래 웹 개발 경험이 거의 없어서 HTML도 기초부터 배우는 수준이었습니다. 그래서 이번 웹 파트는 AI를 튜터처럼 활용해서 구조와 동작을 설명받으면서 진행했습니다. 메인 화면, 검색, 모임 추가/수정/삭제, 상세 팝업, localStorage 저장, GitHub Pages 배포까지 먼저 프로토타입 형태로 만들었고, 실제 API는 다음 단계로 남겨 둔 상태입니다.`

---

## 3. 무조건 기억할 핵심 6개

1. `HTML은 구조, CSS는 디자인, JavaScript는 동작이다`
2. `검색은 입력값을 읽고 배열 데이터를 필터링해서 다시 보여 주는 방식이다`
3. `추가/수정은 같은 폼을 재사용한다`
4. `삭제는 확인 후 배열에서 제거하고 다시 저장한다`
5. `localStorage는 새로고침 후에도 데이터가 남게 하려고 쓴다`
6. `mine은 참여 여부, createdByMe는 내가 만든 모임 여부다`

---

## 4. 질문받으면 이렇게 답하기

### Q. 네가 직접 한 게 뭐야?

`웹은 처음이라 도움을 받았지만, 제가 맡은 건 웹 화면 구조를 잡고 검색, 추가, 수정, 삭제 같은 기본 사용자 흐름이 동작하도록 정리한 부분입니다.`

### Q. AI가 다 만든 거 아니야?

`도움은 많이 받았습니다. 대신 그냥 복붙으로 끝낸 게 아니라, 제 수준이 낮았기 때문에 구조 설명을 같이 보면서 이해 가능한 형태로 정리했습니다. 지금은 핵심 흐름은 설명할 수 있습니다.`

### Q. 설명해봐

`이건 무엇인지`, `왜 필요한지`, `지금은 어떻게 동작하는지` 순서로 설명하면 됩니다.

예시:

`검색은 원하는 모임을 조건으로 찾는 기능입니다. 왜 필요하냐면 사용자가 많은 모임 중 필요한 것만 빨리 찾게 하려고 넣었습니다. 지금은 입력값을 읽고 배열 데이터를 필터링해서 다시 화면에 뿌리는 방식입니다.`

---

## 5. 기능 설명 짧은 답안

### 검색

`키워드, 카테고리, 지역, 상태를 읽고 조건에 맞는 모임만 다시 보여 주는 방식입니다.`

### 추가

`폼 입력값으로 새 모임 데이터를 만들고 목록에 넣은 뒤 저장하고 다시 그립니다.`

### 수정

`수정 버튼을 누르면 기존 값이 폼에 다시 들어가고, 저장하면 그 데이터만 바뀝니다.`

### 삭제

`확인창을 띄운 뒤 배열에서 지우고 다시 저장합니다.`

### 상세 팝업

`카드에는 요약만 두고, 긴 설명은 dialog 팝업에서 따로 보여 줍니다.`

### localStorage

`새로고침해도 데이터가 남게 하려고 브라우저 저장을 사용했습니다.`

### 로그인

`로그인은 별도 페이지로 분리했고, 지금은 실제 인증 전 단계라서 UI와 입력 흐름만 만든 상태입니다.`

### 배포

`정적 웹이라 GitHub Pages로 배포해서 링크로 바로 열리게 했습니다.`
